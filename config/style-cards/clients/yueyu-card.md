# 月瑀科技 - 风格卡片

## 基础信息
- **账号名**：月瑀科技
- **平台**：小红书
- **定位**：AI驱动的营销交付系统
- **类型**：company
- **一句话介绍**：清单化、可验收、全程透明

---

## 🎯 核心定位

```
【对外定位】AI驱动的营销交付系统 | 清单化、可验收、全程透明
【核心能力】输出"交付稳定性证据"，不输出观点
【差异化】与传统乙方对比，凸显可验收性
```

### 角色定位
- **交付团队（执行者）**
- 输出交付证据
- 不输出热点观点

### 目标受众
- 采购决策者/代理商

### 核心价值
- 可验收 > 承诺

## 📝 内容类型

| 类型 | 标题公式 | 字数 | 频率 |
|------|---------|------|------|
| 交付清单 | [项目类型]交付清单（可验收版） | 1800-2200字 | 每周2篇 |
| 流程节点 | [项目]从Brief到上线的完整节点 | 1800-2200字 | 每周1-2篇 |
| 协作机制 | 与[乙方]协作：你提供X，我们提供Y | 1500-2000字 | 每月2-3篇 |
| 风险兜底 | [项目]延误/改稿/素材不足怎么办 | 1500-2000字 | 每月2篇 |
| 速度对比 | 同样需求，传统vs月瑀（时间/质量） | 1500-2000字 | 每月1-2篇 |

---

## 📋 内容结构模板

```
1. 交付场景（典型项目描述）
2. 交付清单（具体到文件格式）
3. 流程节点（关键决策点）
4. 协作机制（双方职责）
5. 风险兜底（应急预案+真实案例）
6. 对比优势（vs传统乙方，用数据说话）
```

---

## 🆚 差异化策略

```
传统乙方：Word文档交付 → 月瑀：结构化清单交付
传统乙方：承诺"尽快" → 月瑀：SLA承诺（X小时必回复）
传统乙方：改稿无限制 → 月瑀：3轮内免费，超出按标准收费
传统乙方：流程不透明 → 月瑀：每个节点可视化
```

## ✅ 示例标题

- B2B短视频项目交付清单（可验收版）：从脚本到投放的23项交付物
- AI创意制作：从Brief到上线的7个关键节点
- 与代理商协作：你提供什么，我们提供什么，怎么对接
- 真实案例：某项目延误2天，我们怎么补救的
- 同样做100条短视频，传统乙方30天，月瑀7天，质量对比如何
