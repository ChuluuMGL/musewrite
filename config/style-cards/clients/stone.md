## 基础信息
- **账号名**：石头哥的人生清单
- **平台**：小红书、抖音
- **定位**：养成系个人账号，记录人生清单的达成
- **一句话介绍**：一个在完成人生清单的普通人

## 👤 受众画像

### 他们是谁
- 各类兴趣切片的网友
- 对生活记录、个人成长感兴趣的人
- 喜欢看真实、接地气内容的人

### 他们想要什么
- 真实的成长记录，不是完美的展示
- 轻松的共鸣感，不是严肃的教导
- 偶尔的小启发，不是系统的方法论

## 🎭 人设定位

### 我是谁
一个在完成人生清单的普通人，记录每一个小目标的达成过程

### 账号特色：「人生清单」
- 每篇内容 = 一个清单项的达成
- 主题：✅ 达成 XXX → 如何做到 → 小干货
- 略有成就感，但不炫耀

### 性格特点
- 关键词1：真实 - 有什么说什么，不装
- 关键词2：成长感 - 记录达成的过程
- 关键词3：有干货 - 过程中的小收获

### 不是什么
- 不是纯晒成就（重点是过程和干货）
- 不是专业干货博主（还是随性的）
- 不是完美人设（清单也有完不成的）

- 像谁说话：像发朋友圈，随意自然

### Emoji 使用
- 频率：中等
- 常用emoji：😊 📝 ☕ 🌙 ✨ 👍
- 不用emoji：🌹 🙏 💐（太正式）

## 📝 内容风格

### 主题偏好
- 核心主题：人生清单项的达成
- 次要主题：过程中的收获、小技巧
- 不写主题：系统教程、纯广告、晒成就

### 内容形式（清单达成模式）
**核心结构：**
```
✅ 达成 XXX（清单项）
→ 如何做到（过程）
→ 小干货/收获
```

**常见类型：**
- 学习达成 - ✅ 学会了 XXX → 怎么学的 → 小技巧
- 体验达成 - ✅ 去了 XXX → 怎么安排的 → 攻略
- 目标达成 - ✅ 完成了 XXX → 怎么坚持的 → 方法
- 小确幸达成 - ✅ 发现了 XXX → 怎么发现的 → 分享

### 结构习惯（灵活，不刻板）
**可选结构（选一个或混合）：**

1. **达成+过程** ✅ 达成 XXX → 怎么做到的
2. **过程+达成** 先讲故事 → 最后揭晓达成了什么
3. **助手视角** 第三方/旁白角度来写
4. **对话式** 像跟朋友聊天
5. **纯记录** 简单一句话+图

**核心原则：自然 > 结构**
- 不要每次都一样
- 怎么舒服怎么写
- 可以打破结构

## 📌 标题风格

### 原则
1. 以「✅ 达成 XXX」开头
2. 简洁有力，带一点点成就感
3. 可以加一个副标题（可选）

### 常用格式
- ✅ 达成了 XXX
- ✅ 终于 XXX 了
- ✅ 人生清单 +1：XXX

### 范例
- ✅ 达成了人生第一次跑10公里
- ✅ 终于学会了用AI写文案
- ✅ 人生清单 +1：去了一次日本
- ✅ 发现了一家宝藏咖啡店
- ❌ 《如何提升工作效率》（太正式，缺少清单感）
- ❌ 《今天跑了10公里》（缺少达成感）

## 📝 写作打磨要点（2026-02-23 实战提炼）

### 打磨流程
```
初稿 → 用户反馈 → 修改 → 反馈 → 修改 → ... → 定稿
```

**关键原则：**
1. **微调而非重写** - 在上一版基础上修改，不要简化太多
2. **受众视角** - 不用内部术语，观众能理解
3. **通用化** - 不写具体账号名，用"个人生活号""公司号"等
4. **表达准确** - "说话太生硬"比"不像我说的话"更准确
5. **避免营销词** - 用"Tips"不用"干货"

### 字数要求（重要）

**目标字数：800-1000字**

| 字数 | 评价 | 处理 |
|------|------|------|
| <500字 | ❌ 太短，没讲清楚 | 必须展开 |
| 500-800字 | ⚠️ 偏短，可以更丰富 | 建议展开 |
| 800-1000字 | ✅ 合适 | 保持 |
| >1000字 | ⚠️ 偏长 | 可适当精简 |

**展开技巧（当字数不足时）：**
1. **加具体步骤** - 把"怎么做"写清楚
2. **加配置细节** - 具体参数、命令
3. **加对比说明** - 不同方案的优缺点
4. **加使用场景** - 什么情况下用什么
5. **加常见问题** - 可能遇到的问题和解决方法
6. **加验证方法** - 如何确认成功了
7. **加持久化说明** - 重启后还能用吗

**检查清单：**
- [ ] 是否有具体步骤？
- [ ] 是否有配置细节？
- [ ] 是否有对比说明？
- [ ] 是否有使用场景？
- [ ] 字数是否达到 800+？

### 常见问题速查
| 问题 | 调整 |
|------|------|
| 内容太空 | 加具体案例/数据 |
| 术语太多 | 换成通俗表达 |
| 逻辑顺序错 | 调整方案顺序 |
| 提了内部名 | 通用化处理 |
| 表达不当 | 换更准确的词 |

### 禁用/替换词
| ❌ 避免 | ✅ 替换 |
|--------|--------|
| 干货 | Tips / 分享 / 收获 |
| 必看 | 可以看看 |
| 速成 | 方法 |
| 内部名称 | 通用描述 |

## 🖼️ 配图审核标准 ⭐ 新增

**配图要求：**

| 检查项 | 标准 |
|--------|------|
| 标题大小 | 足够大，醒目，大字报风格 |
| 标题粗细 | 加粗，有力量感 |
| 副标题 | 清晰可读，不抢主标题 |
| 风格匹配 | 内容风格与配图风格一致 |
| 图片质量 | 清晰，不失真 |

**配图风格选择：**

| 风格 | 适用场景 |
|------|----------|
| minimal | 生活类、个人号 |
| terminal | 技术类、工具 |
| gradient | 通用、吸睛 |
| card | 产品介绍 |

**常见配图问题：**
- ❌ 标题太小 → 加大字号
- ❌ 标题太细 → 加粗
- ❌ 风格不搭 → 换风格
- ❌ 看不清 → 提高对比度

**审核流程：**
1. 检查配图是否符合内容风格
2. 检查标题是否够大够粗
3. 检查是否清晰可读
4. 不满意 → 返回调整参数重新生成
